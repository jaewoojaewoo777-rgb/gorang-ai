# 고랑AI 프로젝트 가이드

> 이 파일은 클로드코드가 매 세션 시작 시 자동으로 읽습니다.
> 작업 전 여기 적힌 규칙을 먼저 확인하세요.

---

## 🔁 세션 기억 규칙 (가장 먼저 실행)

클로드코드는 대화가 끝나면 내용을 잊습니다. 과거 작업을 이어가기 위해 **반드시 아래를 지키세요.**

1. **세션 시작 시**: 프로젝트 루트의 `진행상황.md` 파일을 먼저 읽고, 지난 작업이 어디까지 진행됐는지 파악한 뒤 작업을 시작하세요.

2. **세션 종료 전 (또는 중요한 작업 완료 시)**: 오늘 한 일을 `진행상황.md` 맨 위에 **날짜와 함께 추가**하세요. 다음 형식을 따르세요:
   ```
   ## [2026-XX-XX] 작업 제목
   - 한 일: (무엇을 했는지)
   - 변경된 파일: (수정/생성한 파일 경로)
   - 결과: (성공/실패, 테스트 여부)
   - 다음 할 일: (이어서 해야 할 것)
   ```

3. 사용자가 "오늘 뭐 했었지?", "어디까지 했어?" 라고 물으면 `진행상황.md`를 읽고 답하세요.

> ⚠️ `진행상황.md`가 없으면 새로 만드세요. 이게 클로드코드의 "기억"입니다.

## 작업 흐름

1. **수정 전**: `curl`로 GitHub 원본 파일 확인
2. **수정 방식**: 부분 패치보다 **전체 파일 재작성** 선호
3. **배포**: GitHub 푸시 → Vercel/Railway 자동 배포
4. **검증**: 실제 테스트 후 결과 보고
5. **에러 디버깅**: Railway 로그 기반으로 반복 수정, 여러 번 실패 시 완전히 새 파일 제공
